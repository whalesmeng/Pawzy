## 三、游戏玩法设计

### 3.1 核心机制

```
棋盘规格：6x6 / 7x7 / 8x8（根据关卡难度）
消除规则：3个及以上相同元素连成一线即可消除
操作方式：滑动交换相邻元素
```

### 3.2 游戏元素

| 元素类型 | 说明 | 图标建议 |
|----------|------|----------|
| 基础元素 | 5-6种颜色的狗狗头像 | 柴犬、柯基、金毛、哈士奇、泰迪、萨摩 |
| 障碍物 | 冰块、木箱、铁链 | 需要消除多次或特殊方式消除 |
| 收集物 | 狗粮、骨头、玩具球 | 关卡目标物品 |

### 3.3 特殊道具（4连/5连/L形/T形生成）

| 道具 | 生成条件 | 效果 |
|------|----------|------|
| 狗爪炸弹 | 4连消 | 消除周围3x3范围 |
| 彩虹骨头 | 5连消 | 消除全场同色元素 |
| 横向狗绳 | 横向4连 | 消除整行 |
| 纵向狗绳 | 纵向4连 | 消除整列 |
| 超级狗爪 | 两个特殊道具组合 | 大范围爆炸 |

### 3.4 关卡目标类型

| 类型 | 说明 |
|------|------|
| 分数关 | 在限定步数内达到目标分数 |
| 收集关 | 收集指定数量的特定元素 |
| 障碍关 | 清除所有障碍物 |
| 掉落关 | 将特定物品掉落到底部 |
| 混合关 | 以上目标的组合 |

## 四、系统模块设计

### 4.1 关卡系统

```
总关卡数：100+ 关（初版50关）
难度曲线：每10关一个难度阶梯
星级评价：1-3星（根据剩余步数/分数）
关卡地图：线性地图，解锁制
```

**关卡配置示例（JSON）：**
```json
{
  "levelId": 1,
  "boardSize": [6, 6],
  "moves": 20,
  "targetType": "score",
  "targetValue": 1000,
  "elements": ["shiba", "corgi", "golden", "husky"],
  "obstacles": [],
  "starThresholds": [1000, 1500, 2000]
}
```

### 4.2 经济系统

| 货币 | 获取方式 | 用途 |
|------|----------|------|
| 金币 | 通关奖励、每日签到、看广告 | 购买道具、续命 |
| 钻石 | 成就奖励、内购 | 购买金币、解锁关卡 |

### 4.3 道具系统

| 道具 | 效果 | 价格 |
|------|------|------|
| 锤子 | 消除任意单个元素 | 50金币 |
| 刷新 | 重新打乱棋盘 | 100金币 |
| +5步 | 增加5步 | 150金币 |
| 提示 | 显示可消除位置 | 免费（CD 30秒） |

### 4.4 成长激励系统

| 系统 | 说明 |
|------|------|
| 每日签到 | 连续7天递增奖励 |
| 每日任务 | 完成3个任务获得额外奖励 |
| 成就系统 | 累计消除、通关等成就 |
| 排行榜 | 好友/全服排行（可选） |

## 十二、注意事项

1. **版权问题**：避免使用与 Candy Crush 等知名游戏相似的美术风格
2. **隐私政策**：上架需准备隐私政策页面
3. **广告合规**：激励视频需明确告知用户
4. **数据存储**：本地存档需加密，防止玩家修改
5. **性能优化**：低端机适配，控制内存占用
